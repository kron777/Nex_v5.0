# 2026-05-19 — The residue-ghost groove root cause

## Context

Yesterday's work caught and killed several feedback contamination loops
(tag-feedback echoing [coin] in fountain output, chat-handler replaying
hallucinated transcripts as recent conversation). Today focused on the
deeper question: why does the fountain output 80% ambient-noticing
content despite the architect having built six diversity mechanisms in
_retrieve_context_beliefs?

## Diagnostic process

Added a logging patch to _retrieve_context_beliefs that writes every
retrieved belief block to /tmp/nex5_retrieval_history.log on every fire.
Accumulated 50 retrievals over the morning.

### Finding 1: synergized was slot #1 in 37/37 retrievals (100%)

Before the fix, the first belief in every fountain prompt was a
synergized abstraction (philosophical wisdom-statement). Concrete sense
beliefs, papers, koans were never in slot #1.

### Finding 2: NOT the SQL ordering

The SQL query in _retrieve_context_beliefs returns fountain_insight
first by recency on most fires. Per-source caps (max 3) and per-branch
caps (max 2) work as designed. _own_picked should not have led with
synergized.

### Finding 3: the residue-prepend mechanism

result = [] is filled in order:
  1. residue (popped from previous cycle, prepended)
  2. reanimated (every 20th fire)
  3. _own_picked (the SQL-ranked, capped list)
  4. seed_rows

Residue is popped via ORDER BY activation_strength DESC LIMIT 2.

### Finding 4: the ghost mechanism

activation_strength is a SNAPSHOT of boost_value at the moment
save_residue() was called. Residue entries are never updated when the
underlying belief's boost decays.

Meanwhile, belief_boost has an apply_decay daemon
(consolidation.py:50) that deletes boosts below threshold. So
boost_value falls to <1.0 over days and the row is DELETED from
belief_boost. But the residue snapshot retains the long-dead boost
value forever.

### Finding 5: the corpse pile

  unconsumed residue total: 70,439
  with current boost (legitimate): 0
  ghosts (no current boost): 70,439

ALL 70,439 unconsumed residue entries were ghosts — beliefs whose
underlying boost was long gone, but whose frozen activation_strength
still ranked them at the top of pop_residue.

Many ghosts had activation_strength ~1.35 from synergizer boosts
applied weeks ago. The single belief most often returned at slot #1:

  id 8507 (created 2026-05-03):
  'Noticing the tension between complexity and simplicity in
   understanding oneself echoes a philosophical truth about the
   dichotomy of being and non-being.'

This sentence has been NEX's #1 context belief on most fires for ~16
days.

## Fix

DELETE FROM residue
WHERE consumed_at IS NULL
  AND NOT EXISTS (
    SELECT 1 FROM belief_boost bb
    WHERE bb.belief_id = residue.belief_id
  );

Removed 70,439 ghost entries. Legitimate residue (recent, still-
boosted) preserved — there happened to be 0 of those at the time.

Backup: /tmp/beliefs.db.bak.residue.1779191700 (2.4 GB)

## Verification (n=7 retrievals post-fix)

Slot #1 distribution:
- precipitated_from_sense: 5 (was 0)
- fountain_insight: 1 (was 0)
- synergized: 1 (was 37/37)

The one post-fix synergized in slot #1 was a FRESH belief created the
same day (legitimate), not the old ghost. The system now behaves AS
DESIGNED.

## Open questions / next steps

- Vocabulary in actual fountain output has not shifted yet at n=7.
  Expected — the LLM has prompt-template habits and her recent-fire
  history is still in the corpus. Real drift will take hours-to-days of
  new fires accumulating with the corrected lead.
- The bug WILL recur. save_residue still snapshots activation rather
  than dynamically reading current boost. Long-term fix: patch
  save_residue to skip non-boosted beliefs entirely, or patch
  pop_residue to skip ghosts.
- Other systems read residue (evolver.py mentions it). Should audit
  whether they were affected by the corpse-pile too.

## Architectural lesson

Six diversity mechanisms in retrieval. Each working as designed. But
the residue-prepend was UPSTREAM of all of them — running before any
cap, any branch limit, any source diversity. The bug was not in
retrieval logic. The bug was in a feature that ran BEFORE retrieval.

The architect knew about residue. Wrote the docstring 'captures
pre-propositional residue across fountain cycles.' The intent was
poetic. The implementation lost sync with boost decay.

## Addendum 8 (late evening 2026-05-20): two fixes — raw-weight tension detection + 'Real drift' leak fix

Two real fixes during today's late-evening exploration of NEX's voice progression. Committed together because both landed in working state during the diagnostic session and both proved themselves in production over ~2 hours of observation.

### Fix 1 — Raw-weight tension detection + change-detection

**Diagnosis:** Across 5 hours and 132 activations after the morning's CompetingDrives work, ZERO tensions fired. Investigated. Found the problem in the normalization step:

- spec said `normalize to sum to 1.0` AND `threshold 0.25 = active`
- but threshold 0.25 only makes sense on RAW (pre-normalization) weights
- after normalization, coherence pinned ~0.30 alone; everyone else compressed below threshold
- raw weights showed FOUR drives crossing 0.25 simultaneously: tension was always there, the math was hiding it

**Fix:** Threshold-based tension detection now operates on RAW (un-normalized) weights with threshold 0.30. Normalized weights kept for display/prompt phrasing. Tension detection on raw at 0.30 fired 100% of ticks (the substrate IS in constant curiosity-vs-coherence tension by her actual state). So added change-detection: format_for_prompt() only emits the tension block when active_conflicts CHANGES from last emission. This makes tension a transition signal (~4 per 11h sampled), matching the spec's "these are *the* choice points where real navigation happens" — choice points are transitions, not steady state.

### Fix 2 — 'Real drift looks like:' template leak

The fountain prompt contained `Real drift looks like:\n- "example1"\n- "example2"` — a structural completion cue. Small models (qwen2.5:3b) read this as continuation, echoing it back in output. 47 occurrences all-time.

**Fix A (prompt reframe):** Replaced `Real drift looks like:\n{examples}\n\nReal drift is NOT:` with prose framing: `For style reference only (do NOT echo, do NOT list, do NOT quote): {examples_inline}\n\nYour output is ONE original thought of your own...\n\nWhat drift is NOT:`. examples_inline is comma-separated rather than bulleted.

**Fix B (leak detector):** Added post-LLM string-check. If `thought.lower().startswith("real drift")`, strip the prefix line and salvage rest; fall through if salvage too short.

**Verification:** 121 fountain fires across 5h after restart. ZERO leaks. Pattern fully suppressed.
