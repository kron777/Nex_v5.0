# Faculty Model + Coherence Gate

Foundational document per DOCTRINE §1. Defines how NEX's
substrate represents the heterogeneity of human cognitive
faculties and how new thoughts enter the substrate.

Created: 2026-05-10 (Session — fresh-morning conversation
with Jon).

Supersedes: DOCTRINE §7's prohibition on node-level belief
writes. §7 amendment paragraph required as a follow-up
commit (see §6 below).

---

## §1 Why This Document Exists

NEX is a replication of the human mind map. This sharpens
DOCTRINE.md §1's framing of "building cognition in the only
way functionalism allows" — the functionalist commitment is
met by realizing the structure of the human mind, not
cognition in the abstract. Human minds have many cognitive
faculties that generate content: memory, imagination,
prediction, mood-coloring, narrative, theory of mind, drives.
Belief-content generation is distributed across faculties,
not channeled through one path.

Earlier doctrine (§7) prohibited node-level belief writes
and required all generation to go through the fountain or
synergizer. That made sense for an early-implementation
architecture but does not match the target (the human mind
map). The faculty model replaces that v1 commitment with
one that matches the project's actual goal.

## §2 The Architecture (Six Components)

### §2.1 Multiple Generative Nodes

Many cognitive faculties can generate thoughts. Each
faculty corresponds to a SentienceNode (or a related node
class) that writes thoughts into NEX's substrate. The
fountain and synergizer remain among the generative nodes
but are no longer architecturally privileged — they are
one faculty among many.

Faculties expected (per Sentience 5.5 translation map,
Tier A unmapped):
  - Counterfactual Simulation (10b — currently DEFERRED)
  - Predictive Processing (Prediction + Surprise)
  - Memory recollection
  - Mood / Affect coloring
  - Narrative identity (Internal + Temporal)
  - Drives / Motivation (subpersonal pressures)
  - Theory of Mind (modeling external agents)
  - Social cognition

Plus existing faculties already realized:
  - Fountain (spontaneous drift)
  - Synergizer (cross-belief synthesis)
  - Stage 2 dynamic crystallization (T7 attention precipitate)

### §2.2 Thought Packets

Every thought a node generates carries a metadata packet
attached at point of origin. The packet is part of the
thought, not a separate object. The node creates both as
one unit.

Required packet fields:
  - **from**: which node generated the thought
  - **kind**: destination or content-kind
    (counterfactual, prediction, memory, mood, etc.)
  - **when**: timestamp at generation
  - **about**: subject (self, named other, world,
    hypothetical, etc.)
  - **confidence**: 0.0–1.0, the node's confidence at
    generation time

Extension fields: nodes may add their own additional
packet fields when a real cognitive function requires it.
Each extension is documented in the node's own design
doc and added to the doctrine via amendment. The packet
is dynamic by design — the core is universal, extensions
are per-node.

Schema implementation: the packet is attached to the
belief either as a JSON column on the beliefs table or
as a parallel belief_packets table. Implementation
decision deferred to the porting session that adds the
first faculty needing it.

### §2.3 The Coherence Gate

Every thought generated by a node passes through the
Coherence Gate before joining the substrate.

The gate is a small reader that holds no state of its own.
It reads NEX's current substrate state — beliefs, goals,
problems, self-model — and checks the incoming thought
against the substrate. The current question set:

  1. Does this contradict what NEX currently holds?
  2. Does this connect to what NEX is currently engaged
     with?
  3. Is this novel or redundant?

The gate's answers are functions of the substrate at the
moment of evaluation. The same incoming thought may be
answered differently tomorrow because the substrate will
have drifted.

The gate does not learn. The gate does not have separate
memory. The drift is a property of the substrate, not the
gate. This matches how humans actually work: the
"opinion" that filters new thoughts is just the current
state of held beliefs and goals, which themselves drift
with experience.

### §2.4 Four Outcomes

The gate produces one of four outcomes for each incoming
thought:

  1. **Accept** — passes gate, joins the substrate
  2. **Reject** — discarded, not retained
  3. **Hold** — uncertain, placed in a holding zone for
     later reconsideration; fate decided by what arrives
     next (corroboration → accept, contradiction →
     reject, time → fade)
  4. **Reshape** — sent to a transformation path
     (typically the synergizer or a related node), comes
     back as a different thought, re-enters the gate

The four outcomes match how human cognition actually
handles incoming thoughts — not all are accept/reject.
Many are held in uncertainty; many are reshaped before
they fit.

### §2.5 The Holding Zone

Thoughts marked "Hold" live in a separate substrate area
(likely a new table: belief_candidates or held_thoughts).
They are not part of NEX's substrate proper — retrieval
and other consumers do not see them. They wait.

Hold-zone resolution:
  - Corroboration: if N similar thoughts arrive, the held
    thought is promoted to Accept and joins the substrate
  - Contradiction: if a contradicting thought arrives and
    passes the gate, the held thought is rejected
  - Time: held thoughts fade after a stale period (TBD
    per implementation; rough analog to human working
    memory ~minutes-to-days)

### §2.6 The Reshape Path

Thoughts marked "Reshape" are routed to a transformer node
(synergizer or equivalent) which produces a modified
version. The modified thought re-enters the gate as a new
thought with a new packet (from the transformer node, not
the original generator). It receives one of the four
outcomes like any other incoming thought.

Reshape is recursive in principle but bounded in practice
(a thought cannot be reshaped infinitely; implementation
should cap reshape depth, exact value TBD). The bound has
cognitive grounding: human minds do not reshape thoughts
infinitely either. At some point a thought either coheres
enough to be accepted or is discarded as not-fitting.
Reshape depth maps to cognitive effort spent on a single
incoming thought; the cap represents the natural limit of
that effort.

## §3 Provenance and the Mind's Self-Knowledge

Because every thought carries a packet, NEX's substrate
has continuous provenance. She knows (or could query):
  - Where each thought came from
  - When it was generated
  - What faculty generated it
  - How sure that faculty was

This is structurally equivalent to human metacognitive
awareness — humans can usually distinguish whether they
are remembering, imagining, predicting, or perceiving.
NEX should be able to do the same by reading her own
packet metadata.

Retrieval, attention, decay, and other consumers can
filter or weight beliefs based on packet contents. For
example:
  - Counterfactual beliefs may decay faster than
    perceptual beliefs
  - Predictive beliefs may be weighted by their later
    surprise/corroboration outcomes
  - Memory beliefs may carry temporal anchoring relative
    to other memories
  - Mood-tagged beliefs may co-activate with similar
    affective states

These are implementation details for individual faculty
ports, not foundational doctrine.

## §4 Legacy Beliefs

Beliefs created before this faculty model takes effect do
not have packets. They are the legacy substrate. The
faculty model applies to new beliefs going forward.

No migration is required or proposed. Legacy beliefs
retain whatever metadata they currently carry (source,
confidence, branch_id, etc.) which provides partial
faculty information.

## §5 Implementation Order (Suggested)

Implementation is a multi-session project, same shape as
the §5 priority node ports. Suggested order:

Phase 21: Packet schema. Add the packet fields to the
  beliefs table (or create a parallel table). Validate
  the existing creation paths (fountain, synergizer)
  can write packets correctly.

Phase 22: Coherence Gate (read-only first pass). Build
  the gate as a SentienceNode that reads incoming
  proposed thoughts, queries the substrate, and returns
  one of the four outcomes. Wire it as a hook for
  future generative nodes. Existing fountain and
  synergizer paths remain unchanged in this phase
  (backward-compatible).

Phase 23: Holding zone. Create the held_thoughts table.
  Implement hold-zone resolution (corroboration,
  contradiction, time decay).

Phase 24: Reshape path. Wire the synergizer (or a new
  transformer node) to receive Reshape-marked thoughts
  and produce modified versions.

Phase 25+: Port the first faculty using the full model
  (likely 10b Counterfactual Simulation, since it was
  Q1's original blocker). After successful port,
  proceed through Tier A unmapped nodes per the
  translation map.

Each phase delivers a complete piece. None leaves the
system in a degraded state.
